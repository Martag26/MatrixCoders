Git y GitHub

Instalación y configuración global de Git

Instalación

  • Windows: descarga el instalador en https://git-scm.com
  • macOS: brew install git o viene con Xcode Command Line Tools
  • Linux (Debian/Ubuntu): sudo apt install git
# Verificar instalación
git --version   # git version 2.42.0

Configuración global (lo primero que haces)

# Identidad — aparece en cada commit
git config --global user.name  "Tu Nombre"
git config --global user.email "tu@email.com"

# Editor por defecto (VS Code)
git config --global core.editor "code --wait"

# Rama principal llamada 'main' (recomendado)
git config --global init.defaultBranch main

# Ver toda la configuración
git config --list

Iniciar un repositorio

# En un proyecto existente
cd mi-proyecto
git init            # crea carpeta .git oculta

# Flujo básico
git status          # ver estado de archivos
git add .           # añadir todo al staging
git add archivo.js  # añadir archivo concreto
git commit -m "Primer commit"

El archivo .gitignore

# .gitignore — archivos que Git debe ignorar
node_modules/
.env
*.log
dist/
.DS_Store
*.tmp

Usa https://gitignore.io para generar un .gitignore según tu stack (Node, Python, Java, etc.).

Comandos esenciales del día a día

ComandoAcción
git statusVer archivos modificados/staging
git add .Añadir todos los cambios al staging
git commit -m "msg"Guardar snapshot en el historial
git log --onelineVer historial resumido
git diffVer cambios sin staged
git diff --stagedVer cambios en staging
💡 Escribe mensajes de commit en imperativo presente: "Add login feature" no "Added login feature". Sé descriptivo: el porqué es más valioso que el qué.
