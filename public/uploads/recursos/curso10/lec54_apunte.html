Git y GitHub

Clonación de repositorios y gestión de Tags

Clonar un repositorio

# Clonar desde GitHub
git clone https://github.com/usuario/repo.git

# Clonar con nombre de carpeta personalizado
git clone https://github.com/usuario/repo.git mi-carpeta

# Clonar solo la rama main (más rápido para repos grandes)
git clone --branch main --single-branch https://github.com/usuario/repo.git

Historial y viaje en el tiempo

# Ver historial detallado
git log

# Historial resumido con árbol de ramas
git log --oneline --graph --all

# Ver qué cambió en un commit
git show abc1234

# Volver a ver el estado en un commit pasado (modo lectura)
git checkout abc1234

# Volver a main
git checkout main

Tags — marcar versiones

# Tag anotado (recomendado para releases)
git tag -a v1.0.0 -m "Primera versión estable"

# Tag ligero (solo un marcador)
git tag v1.0.0-beta

# Ver todos los tags
git tag

# Subir tags a GitHub
git push origin v1.0.0
git push origin --tags    # subir todos

# Eliminar tag
git tag -d v1.0.0-beta
git push origin --delete v1.0.0-beta

Semantic Versioning (SemVer)

FormatoCuándo incrementar
MAJOR (1.x.x)Cambios incompatibles con versiones anteriores
MINOR (x.1.x)Nueva funcionalidad compatible hacia atrás
PATCH (x.x.1)Corrección de bugs compatible
💡 GitHub convierte automáticamente los tags en Releases. Añade notas de la versión, changelogs y archivos binarios desde la interfaz web.
