Configuración y sincronización de proyectos en VS Code

Git integrado en VS Code

VS Code incluye soporte nativo para Git sin necesidad de instalar extensiones. El panel Source Control (Ctrl+Shift+G) muestra todos los cambios.

Operaciones básicas desde la UI

AcciónEn VS Code
git statusPanel Source Control — lista de archivos cambiados
git addClic en + junto al archivo o "Stage All Changes"
git commitEscribe el mensaje y pulsa Ctrl+Enter
git pushBotón "Sync Changes" o menú ··· → Push
git pullBotón "Sync Changes" o menú ··· → Pull
git logExtensión GitLens o menú ··· → View History

Terminal integrado de VS Code

# Abrir terminal: Ctrl + `
git status
git add .
git commit -m "feat: add dark mode"
git push origin main

Extensiones recomendadas para Git

  • GitLens: supercharged git — quién cambió qué línea, historial visual
  • Git Graph: árbol visual del historial de commits y ramas
  • GitHub Pull Requests: gestiona PRs directamente desde VS Code

Resolver conflictos en VS Code

VS Code muestra un editor visual con botones para elegir entre "Aceptar actual", "Aceptar entrante", "Aceptar ambos" o editar manualmente. Mucho más cómodo que editar el texto con los marcadores.

💡 Usa la barra de estado inferior de VS Code — muestra la rama actual, el número de commits pendientes de push/pull y permite cambiar de rama con un clic.
