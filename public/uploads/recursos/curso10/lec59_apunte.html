Gestión de ramas y trabajo remoto desde VS Code

Gestión de ramas desde VS Code

# Desde la barra de estado inferior:
# Clic en el nombre de la rama → menú de ramas

# Opciones disponibles:
# • Create new branch
# • Create new branch from...
# • Checkout branch (cambiar)
# • Delete branch
# • Rename branch

Flujo de trabajo típico en VS Code

  1. Clic en rama en la barra inferior → Create new branch → feature/nueva-funcion
  2. Desarrolla y guarda los cambios
  3. Panel Source Control → Staging → Commit
  4. Menú ··· → Publish Branch (primera vez) o Push
  5. En GitHub → Compare & pull request
  6. Tras el merge → vuelve a main → Pull para sincronizar

Fetch vs Pull desde VS Code

# Fetch: descarga cambios del remoto SIN fusionar
# Útil para ver qué hay sin afectar tu trabajo
# Panel Source Control → ··· → Fetch

# Pull: descarga Y fusiona (puede crear conflictos)
# Botón Sync Changes o ··· → Pull

Configurar origin desde VS Code

# Si el repo no tiene remoto aún, VS Code pregunta al hacer Push
# También puedes hacerlo desde la terminal integrada:
git remote add origin https://github.com/usuario/repo.git
git push -u origin main

GitHub Copilot y VS Code

VS Code integra GitHub Copilot para sugerencias de código con IA. También puedes ver Issues y Pull Requests de GitHub directamente desde el editor con la extensión GitHub Pull Requests and Issues.

💡 Usa Ctrl+Shift+P → "Git: Checkout to..." para cambiar de rama sin salir del editor.
