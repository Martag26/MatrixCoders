Git y GitHub

Fork para colaborar en repositorios ajenos

¿Qué es un Fork?

Un fork es una copia del repositorio original en tu cuenta de GitHub. Te permite experimentar libremente sin afectar al proyecto original. Cuando quieres contribuir, abres un Pull Request desde tu fork.

Flujo de contribución Open Source

1. Fork → Tu copia en GitHub 2. Clone → Tu máquina local 3. Branch → Crea rama para el cambio 4. Commit → Desarrolla y commitea 5. Push → Sube a tu fork en GitHub 6. Pull Request → Al repo original 7. Review → Mantenedores revisan 8. Merge → Tu código entra en el proyecto

Comandos del flujo de fork

# 1. Clona TU fork (no el original)
git clone https://github.com/TU_USUARIO/repo.git

# 2. Añade el repo original como remoto "upstream"
git remote add upstream https://github.com/AUTOR_ORIGINAL/repo.git

# 3. Crea una rama para tu contribución
git checkout -b fix/typo-in-readme

# 4. Haz tus cambios y commitea
git add .
git commit -m "fix: correct typo in README"

# 5. Sube a tu fork
git push origin fix/typo-in-readme

# 6. En GitHub → botón "Compare & pull request"

Mantener tu fork actualizado

# Descargar cambios del repo original
git fetch upstream

# Fusionar en tu rama main local
git checkout main
git merge upstream/main

# Actualizar tu fork en GitHub
git push origin main

Issues y GitHub Discussions

  • Issues: reportar bugs, sugerir features, hacer preguntas
  • Labels: bug, enhancement, good first issue (ideal para empezar)
  • Milestones: agrupar issues por versión o sprint
  • Discussions: conversaciones abiertas sin ser Issues formales

GitHub Actions — CI/CD básico

# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm install
      - run: npm test
💡 Busca repositorios con la etiqueta good first issue en GitHub para hacer tu primera contribución Open Source. Es la mejor forma de aprender Git en un entorno real.
