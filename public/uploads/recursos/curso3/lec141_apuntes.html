Despliegue y buenas prácticas

matrixCoders · Apuntes de lección · Backend con Node.js y Bases de Datos

PM2 — Gestor de procesos

# Instalar globalmente
npm install -g pm2

# Iniciar aplicación
pm2 start server.js --name "mi-api" --instances max

# Comandos útiles
pm2 status
pm2 logs mi-api
pm2 restart mi-api
pm2 save               # persistir entre reinicios

# ecosystem.config.js
module.exports = {
  apps: [{
    name: "mi-api",
    script: "server.js",
    instances: "max",
    exec_mode: "cluster",
    env_production: { NODE_ENV: "production", PORT: 80 }
  }]
};

Docker

# Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY src ./src
COPY server.js .
EXPOSE 3000
USER node
CMD ["node", "server.js"]

# docker-compose.yml
services:
  api:
    build: .
    ports: ["3000:3000"]
    env_file: .env
    depends_on: [db]
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: miapp
      POSTGRES_PASSWORD: secreto

Checklist de buenas prácticas

  • Usar Helmet.js para cabeceras HTTP seguras
  • Limitar tamaño del body (express.json({ limit: "10kb" }))
  • Rate limiting con express-rate-limit
  • Variables de entorno en .env, nunca en el código
  • Validar siempre la entrada del usuario (Joi / Zod)
  • Logs estructurados con Pino o Winston
  • Tests unitarios (Jest) + tests de integración (Supertest)
  • Manejo centralizado de errores con middleware
  • HTTPS obligatorio en producción (nginx reverse proxy)
