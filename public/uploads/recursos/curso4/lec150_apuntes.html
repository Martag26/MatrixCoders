Despliegue y buenas prácticas

matrixCoders · Apuntes de lección · HTML y CSS

Optimización antes de publicar

  • Imágenes: convierte a .webp o .avif, usa loading="lazy"
  • CSS/JS: minifica con herramientas (esbuild, Vite, Lightning CSS)
  • Fuentes: font-display: swap y limita pesos cargados
  • Cache: cabeceras correctas (estáticos con hash → max-age=31536000, immutable)

Accesibilidad (WCAG)

  • Contraste mínimo: 4.5:1 texto normal, 3:1 texto grande
  • Atajos de teclado funcionando, foco visible
  • Etiquetas ARIA solo cuando HTML semántico no basta
  • Test con lectores de pantalla: NVDA, VoiceOver
  • Auditoría: Lighthouse, axe DevTools

Hosting estático gratuito

  • GitHub Pages — gratis, ideal para portfolios
  • Netlify / Vercel — despliegue automático desde git, HTTPS gratis
  • Cloudflare Pages — CDN global incluido
# Deploy típico en Netlify
git init
git add .
git commit -m "v1"
git push origin main
# Netlify detecta el push y publica automáticamente

Checklist final antes de publicar

  • ✅ Páginas validan en validator.w3.org
  • ✅ Funciona en móvil, tablet y desktop
  • ✅ Lighthouse > 90 en Performance, Accessibility, SEO, Best Practices
  • ✅ Favicons + meta tags Open Graph
  • ✅ Página 404 personalizada
  • ✅ HTTPS activado
  • ✅ Formularios con validación HTML y servidor
