Optimización para producción

  • Bundle: npm run build con Vite/Webpack — minifica, hashea, divide en chunks
  • Tree shaking: imports ESM permiten eliminar código no usado
  • Code splitting: import() dinámico → carga bajo demanda
  • Imágenes: .webp, loading="lazy", srcset
  • CDN: sirve estáticos desde edge (Netlify/Vercel/Cloudflare)

Linters y formateadores

# ESLint — detecta errores y malas prácticas
npm install -D eslint
npx eslint --init

# Prettier — formatea consistentemente
npm install -D prettier
echo '{ "singleQuote": false, "semi": true }' > .prettierrc

# En package.json
"scripts": {
  "lint":   "eslint src/",
  "format": "prettier --write src/"
}

Testing con Vitest

// utils/suma.test.js
import { describe, it, expect } from "vitest";
import { suma } from "./suma.js";

describe("suma", () => {
  it("suma dos números positivos", () => {
    expect(suma(2, 3)).toBe(5);
  });

  it("acepta arrays con spread", () => {
    expect(suma(...[1,2,3])).toBe(6);
  });
});

Checklist antes de publicar

  • ✅ Sin console.log en producción (linter rule)
  • ✅ Variables sensibles en .env, jamás en el bundle
  • ✅ Manejo de errores en todas las llamadas fetch
  • ✅ Validación tanto en cliente como en servidor
  • ✅ Lighthouse Performance ≥ 80
  • ✅ Funciona sin JS (al menos contenido visible)
  • ✅ Service Worker o estrategia offline (si aplica)
