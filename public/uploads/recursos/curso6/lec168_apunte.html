Despliegue y buenas prácticas en PHP

Variables de entorno y configuración

# .env (nunca lo subas a git)
DB_HOST=localhost
DB_NAME=mi_bd
DB_USER=root
DB_PASS=secret
APP_ENV=production
<?php
// Leer con phpdotenv o manualmente:
$host = getenv('DB_HOST') ?: 'localhost';

// O en Apache, via SetEnv en .htaccess:
// SetEnv DB_HOST localhost
?>

Seguridad esencial

<?php
// 1. Sanitizar salida HTML
echo htmlspecialchars($input, ENT_QUOTES, 'UTF-8');

// 2. Hashing de contraseñas
$hash = password_hash($password, PASSWORD_BCRYPT);
password_verify($input, $hash); // retorna bool

// 3. Validar y filtrar inputs
$email = filter_var($_POST['email'], FILTER_VALIDATE_EMAIL);
$edad  = filter_var($_GET['edad'],  FILTER_VALIDATE_INT);

// 4. Sesiones seguras
session_start();
session_regenerate_id(true); // previene session fixation
?>

Optimización de consultas

-- Agrega índices a columnas usadas en WHERE/JOIN
CREATE INDEX idx_email ON users(email);

-- Evita SELECT *
SELECT id, nombre, email FROM users WHERE activo = 1;

-- Usa LIMIT para paginar
SELECT * FROM productos LIMIT 10 OFFSET 20;

Estructura de proyecto lista para producción

proyecto/
├── .env               # variables de entorno (en .gitignore)
├── .htaccess          # redirige todo a public/index.php
├── composer.json      # dependencias (autoload, etc.)
├── public/
│   └── index.php      # único punto de entrada
└── app/
    ├── config/
    ├── controllers/
    ├── models/
    ├── views/
    └── helpers/

Checklist de despliegue

  • ✅ display_errors = Off en producción
  • ✅ HTTPS habilitado (certificado SSL)
  • ✅ Contraseñas con bcrypt, nunca en texto plano
  • ✅ Prepared statements en todas las queries
  • ✅ Headers de seguridad (Content-Security-Policy, X-Frame-Options)
  • ✅ Backups automáticos de la base de datos
  • ✅ Variables sensibles en .env, no en el código
💡 Usa Composer para gestionar dependencias y activa el autoloader PSR-4 para no tener que hacer require manual de cada clase.
