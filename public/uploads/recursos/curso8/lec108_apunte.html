Introducción al Estado en React

¿Qué es el estado?

El estado (state) son los datos que pueden cambiar con el tiempo y que, al cambiar, hacen que React re-renderice el componente con los nuevos valores.

Props vs Estado

PropsEstado
¿Quién lo controla?El padreEl propio componente
¿Mutable?No (inmutable)Sí (con setter)
¿Causa re-render?Sí (cuando el padre cambia)Sí (al actualizarse)

Cuándo usar estado

  • Un contador que el usuario incrementa
  • Un campo de formulario que el usuario rellena
  • Si un modal está abierto o cerrado
  • Datos cargados desde una API
  • El tema actual (claro/oscuro)

Antes de Hooks: componentes de clase

class Contador extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }
  render() {
    return <button onClick={() => this.setState({ count: this.state.count + 1 })}>
      {this.state.count}
    </button>;
  }
}
💡 Con Hooks (useState) ya no necesitas clases. Los componentes funcionales con Hooks son más simples y fáciles de leer.
