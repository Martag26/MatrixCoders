Limpieza de efectos en useEffect

¿Por qué limpiar?

Algunos efectos crean recursos que deben liberarse cuando el componente se desmonta o antes de re-ejecutar el efecto: timers, listeners, suscripciones, conexiones WebSocket…

Función de limpieza

useEffect(() => {
  const timer = setInterval(() => {
    console.log('tick');
  }, 1000);

  // Retorna la función de limpieza
  return () => clearInterval(timer);
}, []);

Limpiar event listeners

useEffect(() => {
  const handleResize = () => setWidth(window.innerWidth);
  window.addEventListener('resize', handleResize);

  return () => window.removeEventListener('resize', handleResize);
}, []);

Abortar peticiones fetch

useEffect(() => {
  const controller = new AbortController();

  fetch(url, { signal: controller.signal })
    .then(r => r.json())
    .then(data => setDatos(data))
    .catch(err => {
      if (err.name !== 'AbortError') console.error(err);
    });

  return () => controller.abort();
}, [url]);
💡 Siempre limpia efectos que crean recursos asíncronos. Una petición fetch que se completa después de que el componente se desmonte puede intentar actualizar estado de un componente inexistente.
