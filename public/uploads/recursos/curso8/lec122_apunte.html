Manejo de estados de carga (Loading) y errores

Tres estados de una petición

Toda petición asíncrona pasa por tres estados: cargando, éxito o error. Debes manejar los tres.

Patrón completo

function Producto({ id }) {
  const [producto, setProducto]   = useState(null);
  const [cargando, setCargando]   = useState(true);
  const [error, setError]         = useState(null);

  useEffect(() => {
    setCargando(true);
    setError(null);

    fetch(`https://fakestoreapi.com/products/${id}`)
      .then(r => {
        if (!r.ok) throw new Error(`Error ${r.status}`);
        return r.json();
      })
      .then(data => setProducto(data))
      .catch(e => setError(e.message))
      .finally(() => setCargando(false));
  }, [id]);

  if (cargando) return <div className="spinner">Cargando...</div>;
  if (error)    return <div className="error">⚠️ {error}</div>;
  return <h2>{producto.title}</h2>;
}

Spinner CSS simple

/* spinner.css */
.spinner {
  width: 40px; height: 40px;
  border: 4px solid #e2e8f0;
  border-top-color: #6B8F71;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
@keyframes spin { to { transform: rotate(360deg); } }
⚠️ Comprueba siempre response.ok al usar fetch. Un status 404 o 500 no lanza excepción por defecto — debes hacerlo tú con throw new Error().
