Solución de CORS y diseño de API REST

¿Qué es CORS?

CORS (Cross-Origin Resource Sharing) es un mecanismo de seguridad del navegador que bloquea peticiones desde un dominio diferente al del servidor. Cuando tu frontend en localhost:3000 llama a tu API en localhost:4000, el navegador bloquea la respuesta.

Solución con el paquete cors

npm install cors
const cors = require('cors');

// Permitir todos los orígenes (solo en desarrollo)
app.use(cors());

// Configuración específica para producción
app.use(cors({
  origin: ['https://mifrontend.com', 'http://localhost:3000'],
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization'],
}));

Principios de diseño REST

PrincipioDescripción
Sin estado (stateless)Cada petición es independiente; el servidor no recuerda la anterior
Recursos identificablesCada recurso tiene una URL única: /users/42
Verbos HTTP semánticosGET=leer, POST=crear, PUT/PATCH=actualizar, DELETE=eliminar
RepresentacionesLos recursos se devuelven en JSON (o XML)
Interfaz uniformeConsistencia en nombres de rutas y respuestas

Convenciones de rutas REST

// Plural para colecciones, singular para recurso específico
GET    /api/users          // listar todos
GET    /api/users/42       // obtener uno
POST   /api/users          // crear nuevo
PUT    /api/users/42       // reemplazar completo
PATCH  /api/users/42       // actualizar parcial
DELETE /api/users/42       // eliminar

// Relaciones anidadas
GET    /api/users/42/posts  // posts del usuario 42

Códigos de estado HTTP

// 2xx — Éxito
res.status(200).json(datos);      // OK
res.status(201).json(nuevo);     // Created
res.status(204).send();          // No Content (DELETE)

// 4xx — Error del cliente
res.status(400).json({ error: 'Bad Request' });
res.status(401).json({ error: 'Unauthorized' });
res.status(404).json({ error: 'Not Found' });

// 5xx — Error del servidor
res.status(500).json({ error: 'Internal Server Error' });
💡 Estructura siempre tus respuestas JSON de forma consistente: {"data": ..., "message": "...", "success": true}
