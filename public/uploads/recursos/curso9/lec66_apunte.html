Patrón MVC en Express

En lugar de tener toda la lógica en un archivo, MVC separa responsabilidades en tres capas:

mi-api/
├── src/
│   ├── controllers/       # lógica de negocio
│   │   └── userController.js
│   ├── models/            # acceso a datos
│   │   └── userModel.js
│   ├── routes/            # definición de rutas
│   │   └── userRoutes.js
│   ├── middleware/        # funciones intermedias
│   │   └── auth.js
│   └── app.js             # setup de Express
├── index.js               # punto de entrada
└── package.json

Ejemplo: separar rutas y controladores

// routes/userRoutes.js
const express = require('express');
const router  = express.Router();
const ctrl    = require('../controllers/userController');

router.get('/',    ctrl.getAll);
router.get('/:id', ctrl.getOne);
router.post('/',   ctrl.create);
router.delete('/:id', ctrl.remove);

module.exports = router;

// app.js
app.use('/api/users', require('./routes/userRoutes'));

Middleware propio

// middleware/logger.js
const logger = (req, res, next) => {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next();  // siempre llama a next() para continuar
};
module.exports = logger;

// app.js — aplicar globalmente
app.use(require('./middleware/logger'));

Despliegue en Railway / Render

  1. Sube tu código a GitHub (sin .env ni node_modules)
  2. Conecta el repositorio en Railway.app o Render.com
  3. Configura las variables de entorno en el dashboard
  4. El servicio detecta automáticamente Node y ejecuta npm start
💡 Añade "start": "node index.js" en tu package.json. Las plataformas de despliegue lo buscan automáticamente.
