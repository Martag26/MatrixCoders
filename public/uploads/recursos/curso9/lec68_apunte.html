Tiempo Real con WebSockets y Socket.io

¿Qué son los WebSockets?

HTTP es un protocolo de petición-respuesta: el cliente pregunta, el servidor responde y se cierra la conexión. WebSockets abren una conexión bidireccional persistente entre cliente y servidor, ideal para chat, notificaciones en tiempo real, juegos y dashboards en vivo.

Socket.io — instalación

npm install socket.io
// server.js
const http     = require('http');
const express  = require('express');
const { Server } = require('socket.io');

const app    = express();
const server = http.createServer(app);
const io     = new Server(server, {
  cors: { origin: '*' }
});

io.on('connection', (socket) => {
  console.log('Cliente conectado:', socket.id);

  // Escuchar evento del cliente
  socket.on('mensaje', (data) => {
    console.log('Recibido:', data);
    // Reenviar a todos los clientes conectados
    io.emit('mensaje', data);
  });

  socket.on('disconnect', () => {
    console.log('Cliente desconectado');
  });
});

server.listen(3000);

Cliente (navegador)

<script src="/socket.io/socket.io.js"></script>
<script>
  const socket = io();

  // Enviar al servidor
  document.getElementById('btn').addEventListener('click', () => {
    socket.emit('mensaje', { texto: 'Hola a todos!' });
  });

  // Recibir del servidor
  socket.on('mensaje', (data) => {
    console.log('Nuevo mensaje:', data.texto);
  });
</script>

Salas (rooms) en Socket.io

// Unirse a una sala
socket.join('sala-react');

// Emitir solo a los de una sala
io.to('sala-react').emit('anuncio', 'Nuevo video publicado');

// Emitir a todos excepto el emisor
socket.broadcast.emit('usuario-unido', socket.id);
💡 Socket.io incluye reconexión automática, soporte para rooms y namespaces. Para proyectos simples, la WebSocket API nativa del navegador también es válida.
